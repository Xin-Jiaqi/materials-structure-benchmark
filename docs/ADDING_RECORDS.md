# Adding derived structures

New records are staged outside Git until their ownership, publication, and patent status are clear. A public record must contain the structure file, checksum, provenance, license, controlled classifications, and evidence level.

## Sliding-ferroelectric bilayer

A bilayer is not labelled `sliding-ferroelectric` merely because it was generated by shifting two layers. The tag requires a documented switching relation and non-`not-evaluated` evidence.

```json
{
  "id": "mst-bilayer-example-000001",
  "structure_type": "bilayer",
  "classes": ["computed", "homobilayer", "stacking-state"],
  "phenomena": ["polar", "sliding-ferroelectric"],
  "evidence_level": "dft-verified",
  "relationships": [
    {"type": "parent-monolayer", "target_id": "mst-mono-mc2d2022-002073"},
    {"type": "switching-pair", "target_id": "mst-bilayer-example-000002"}
  ],
  "stacking": {
    "parent_ids": ["mst-mono-mc2d2022-002073", "mst-mono-mc2d2022-002073"],
    "relative_shift_fractional": [0.3333333333, 0.6666666667],
    "interlayer_distance_angstrom": 3.2,
    "registry": "AB",
    "twist_angle_degree": 0.0,
    "generator": {"name": "heterojunction", "version": "record exact release"}
  },
  "ferroelectric": {
    "mechanism": "sliding",
    "state_label": "P+",
    "switching_path_id": "path-example-000001",
    "polarization": {"value": null, "unit": null, "method": "record method"},
    "energy_relative_mev": 0.0,
    "barrier_mev": null,
    "evidence": {"calculation_id": "record reproducible calculation identifier"}
  }
}
```

```json
{
  "id": "mst-intercalated-example-000001",
  "structure_type": "bilayer",
  "classes": ["computed", "intercalated"],
  "phenomena": [],
  "evidence_level": "candidate",
  "relationships": [
    {"type": "host", "target_id": "mst-bilayer-example-000001"}
  ],
  "intercalation": {
    "host_id": "mst-bilayer-example-000001",
    "intercalant_species": ["Li"],
    "concentration": "Li1 per host supercell",
    "site": "hollow",
    "charge_state": null
  }
}
```

The snippets show only the mechanism-specific fields; complete records must also satisfy `schema/material-record.schema.json`. Never place unpublished coordinates in documentation examples.
